Експеримент 4

Display і box-sizing

Перевіряємо, як властивість display змінює поведінку елементів, а box-sizing впливає на розрахунок розміру блоку.

Властивість display

Властивість display визначає, як елемент поводиться у потоці документа: як блок, інлайновий елемент, інлайн-блок або flex-контейнер.

display: block

Block 1
Block 2
Block 3

Блокові елементи займають доступну ширину і розміщуються один під одним.

display: inline

Inline 1 Inline 2 Inline 3

Інлайнові елементи залишаються в одному рядку, доки для них вистачає місця.

display: inline-block

Inline-block 1 Inline-block 2 Inline-block 3

Inline-block поводиться як інлайновий елемент у рядку, але дозволяє задавати ширину, висоту, padding і margin.

display: flex

Flex 1
Flex 2
Flex 3

Flex-контейнер допомагає зручно розташовувати елементи в рядку, керувати відступами та вирівнюванням.

Властивість box-sizing

Властивість box-sizing визначає, як браузер рахує загальний розмір елемента: тільки за контентом або разом із padding та border.

content-box

width: 220px;
padding: 24px;
border: 6px;

При box-sizing: content-box задана ширина стосується тільки контенту. Padding і border додаються зверху.

border-box

width: 220px;
padding: 24px;
border: 6px;

При box-sizing: border-box padding і border вже входять у задану ширину елемента.

Приклад CSS-коду

Нижче показано кілька базових прикладів використання display та box-sizing. 

.display-block-item {
  display: block;
}

.display-inline-item {
  display: inline;
}

.display-inline-block-item {
  display: inline-block;
}

.display-flex-demo {
  display: flex;
}

.border-box-example {
  box-sizing: border-box;
}

Мій висновок

Властивість display допомагає керувати поведінкою елементів у потоці сторінки, а box-sizing робить розрахунок розмірів більш передбачуваним. Найзручнішим варіантом для більшості сучасних макетів є box-sizing: border-box.
